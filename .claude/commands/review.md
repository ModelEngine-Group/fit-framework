审查指定的 Pull Request。

**此命令已迁移到官方插件，提供更专业的多维度审查。**

**用法：**
- `/review` - 基础审查（调用插件）
- `/code-review:code-review <pr-number>` - 深度代码审查（5个并行代理）
- `/pr-review-toolkit:review-pr` - 专业PR审查（6个专业代理）

**推荐使用插件命令：**

### 方案1：深度代码审查（推荐）
```
/code-review:code-review <pr-number>
```
**功能：**
- 5个并行 Sonnet 代理审查
- CLAUDE.md 规范合规性检查
- Bug 检测与历史上下文分析
- 信心分数过滤误判
- 适合：需要全面深度审查时

### 方案2：专业PR审查工具包
```
/pr-review-toolkit:review-pr
```
**功能：**
- 6个专业审查代理，包括：
  - 代码注释准确性分析
  - 测试覆盖质量审查
  - 错误处理检查（静默失败检测）
  - 类型设计质量分析
  - 通用代码质量审查
  - 代码简化优化建议
- 适合：需要多角度专业建议时

**审查维度对比：**

| 审查内容 | code-review | pr-review-toolkit |
|---------|-------------|-------------------|
| CLAUDE.md 合规 | ✅ | ✅ |
| Bug 检测 | ✅ | ✅ |
| 测试覆盖 | ⚠️ | ✅ 专业代理 |
| 错误处理 | ⚠️ | ✅ 静默失败检测 |
| 注释质量 | ⚠️ | ✅ 专业代理 |
| 类型设计 | ⚠️ | ✅ 专业代理 |
| 代码简化 | ⚠️ | ✅ 专业代理 |

**使用建议：**
- 日常PR审查：使用 `/code-review:code-review`
- 重要功能/复杂PR：使用 `/pr-review-toolkit:review-pr`
- 快速检查：直接询问 Claude 具体问题
