根据技术方案实施任务 $1。

执行以下步骤:

1. 验证前置条件:
   - 检查 .ai-workspace/active/$1/task.md 存在
   - 检查 plan.md 存在
   - 读取技术方案,理解实施步骤和文件清单

2. 执行代码实现:
   - 按照 plan.md 中的步骤顺序执行
   - 实现功能代码
   - 编写完整的单元测试
   - 本地运行测试验证功能
   - 更新相关文档和注释
   - 遵循项目编码规范

3. 运行测试验证:
   - Maven 项目: mvn test -pl :{module-name}
   - Node.js 项目: npm test
   - Python 项目: pytest
   - 确保所有测试通过

4. 输出实现报告到 implementation.md,包含:
   - 已修改文件列表(新增文件和修改文件)
   - 关键代码说明(重要逻辑的解释和代码片段)
   - 测试结果(单元测试和集成测试)
   - 与方案的差异(如果有)
   - 待审查事项(需要 reviewer 关注的点)
   - 已知问题
   - 下一步建议

5. 更新任务状态:
   - current_step: implementation
   - updated_at: 当前时间
   - 在工作流进度中标记代码实现为完成

6. 告知用户:
   - 输出实现概要(修改文件数、新增文件数、测试通过情况)
   - 显示输出文件路径
   - 提示下一步使用 /review $1 进行代码审查

**实施原则**:
- 严格遵循方案,不要偏离技术方案
- 分步实施,按照 plan.md 中的步骤顺序执行
- 及时测试,每完成一个步骤就运行测试
- 保持简洁,不要过度设计或添加额外功能

**注意事项**:
- 必须先完成技术方案设计(plan.md 存在)
- 方案必须经过人工审查批准
- 所有新增代码必须有单元测试
- **不要**自动执行 git commit,实现完成后等待代码审查
